# DCC 1.2.0 工作人员测试手册

本手册面向不了解统计学和编程的普通调查工作人员。测试只使用文件夹内的
合成数据，不使用真实受访者数据。至少 1 名真实工作人员即可形成一份有效的
工作人员验证材料；可以增加人数以获得更可靠的改进意见。

工作人员验证用于发现易用性问题，不阻断 DCC 包发布。不得自动生成、代填或
伪造参与者、签名、时间、答案、评分或现场记录。

## 文件夹内容

- `01-安装并完整测试.R`：安装、检查、预览和经授权执行的完整 R 代码。
- `DCC-1.2.0-staff-acceptance.xlsx`：现场记录和评分工作簿。
- `project/`：严格 Excel 计划、等价 JSON 计划和合成 CSV 数据。
- `offline-install/`：DCC 及全部正式输入格式依赖的离线源码仓库。
- `SHA256SUMS`：文件完整性清单。

## 准备

1. 安装 R 4.1 或更高版本。建议安装 RStudio，但不是必须。
2. 将整个测试文件夹解压到普通可写目录，例如桌面；不要只复制其中几个文件。
3. 打开 `DCC-1.2.0-staff-acceptance.xlsx`，主持人为参与者分配 P001。
4. 打开 `project/DCC-cleaning-plan.xlsx`。只查看或编辑黄色单元格，不要增加、
   删除、改名或移动工作表、列和机器表头。

## 第一次运行：只检查和预览

1. 在 RStudio 中打开 `01-安装并完整测试.R`。
2. 确认文件开头是：

   ```r
   AUTHORIZE_EXECUTION <- FALSE
   ```

3. 点击 `Source`。脚本会把 DCC 和正式格式依赖安装到本测试文件夹的
   `R-library/`，不需要管理员权限。
4. 脚本会创建一个新的 `测试输出-日期时间/` 文件夹，并依次完成：
   - 严格计划检查；
   - preview；
   - 原始合成数据 SHA-256 前后核对。
5. 打开 `01-检查/`、`02-预览/` 和 `测试结果摘要.txt`。确认 preview 没有
   执行清洗动作，也没有覆盖 `project/responses.csv`。

每次运行都会使用新的输出目录，不覆盖上次结果。

## 经主持人批准后执行

只有在参与者完成检查、能正确说明 preview 与 execute 的区别，并得到主持人
明确批准后，才执行以下操作：

1. 把脚本开头改为：

   ```r
   AUTHORIZE_EXECUTION <- TRUE
   ```

2. 再次点击 `Source`。
3. 查看新建测试目录中的 `03-执行结果/`。执行结果写入新目录，绝不写回原始
   合成数据文件。
4. 测试结束后，把 `AUTHORIZE_EXECUTION` 改回 `FALSE`。

## 填写工作人员记录

在 `DCC-1.2.0-staff-acceptance.xlsx` 中只填写黄色单元格：

1. `参与者`：日期、主持人、合成数据文件、测试前后 SHA-256 和输出目录。
2. `场景记录`：每个场景的开始、结束、状态、帮助次数、代码编辑、原始覆盖。
3. `区分测试`：由参与者独立回答，主持人不得提示。
4. `SUS问卷`：参与者完成全部 10 项。
5. `签署`：参与者与主持人分别现场确认；不得代签。
6. `评分摘要`：自动计算研究状态。

至少 1 份记录有效，并且完成率与区分正确率均至少 80%、SUS 中位数至少 75、
代码编辑和原始覆盖均为 0 时，工作人员流程状态显示 `pass`。空白材料显示
`facilitator_required`。该状态只描述工作人员研究，不是发布门禁。

## 遇到问题

- 不要修改 R 代码来绕过错误。
- 记录错误出现的时间、完整文字和所在步骤。
- 保留 `错误说明.txt`、`运行日志.txt` 和整个测试输出目录。
- DCC 稳定错误代码可用 `DCC::dcc_help("错误代码")` 查询中文解释。
- 不要在本测试文件夹中放入真实调查数据或个人敏感信息。
