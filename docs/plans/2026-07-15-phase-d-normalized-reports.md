# Phase D Normalized Reports Implementation Plan

> **Execution:** Complete the checked tasks in order, with the stated red/green
> verification and review checkpoint before each commit.

**Goal:** Produce staff, statistical, and machine reports from one normalized,
validated report model so every audience sees the same counts, findings, and
provenance at the appropriate level of detail.

**Architecture:** `dcc_report_model()` converts a completed run into one
versioned, immutable model. Three renderers consume only that model: a concise
staff workbook/HTML report, a reproducible statistical bundle, and a stable
JSON/JSONL machine contract. Renderers never recompute cleaning statistics.

**Tech Stack:** R >= 4.1, data.table, jsonlite, jsonvalidate, openxlsx2,
base R HTML generation, Arrow for Parquet, testthat.

## Global Constraints

- Phases A through C are complete.
- The normalized model is the single source of report truth.
- PDF is neither required nor generated by default.
- No renderer silently truncates rows, findings, labels, or provenance.
- Sensitive examples are redacted by default and require explicit opt-in.
- Renderer failure is recorded without corrupting a successful cleaning run.
- Counts in every renderer must reconcile to the normalized model exactly.
- Before each commit, stage exactly the files listed for that task.

```r
dcc_report_model <- function(result, run = NULL) {
  stopifnot(inherits(result, "dcc_result"))
  model <- list(
    contract = list(name = "dcc-report", version = "1.0"),
    project = report_project(result, run),
    run = report_run(result, run),
    input = report_input(result),
    plan = report_plan(result, run),
    validation = report_validation(result),
    findings = report_findings(result),
    dispositions = report_dispositions(result),
    changes = report_changes(result),
    exclusions = report_exclusions(result),
    reconciliation = report_reconciliation(result),
    summaries = report_summaries(result),
    scoring = report_scoring(result),
    mapping = report_mapping(result),
    provenance = report_provenance(result),
    performance = report_performance(result),
    hashes = report_hashes(result, run),
    outputs = report_outputs(result, run),
    sensitivity = report_sensitivity(result, run)
  )
  validation <- dcc_validate_report_model(model)
  if (dcc_has_errors(validation)) {
    dcc_abort_validation(validation, "Invalid normalized report model")
  }
  structure(model, class = c("dcc_report_model", "list"))
}
```

The helpers copy existing run/result values and may aggregate only once here.
They must not mutate `result` or read a renderer-specific option.

test_that("one run publishes selected audience reports atomically", {
  out <- withr::local_tempdir()
  data <- write_run_csv()
  plan <- valid_plan_workbook(data, outputs = c("staff", "machine"))
  res <- dcc_run(data, plan = plan, output_dir = out)

  expect_true(file.exists(file.path(res$run_dir, "machine", "summary.json")))
  expect_identical(res$manifest$status, "success")

test_that("renderer failure leaves cleaning evidence and failed manifest", {
  local_mocked_bindings(
    dcc_report_staff = function(...) stop("injected renderer failure"),
    .package = "DCC"

  data <- write_run_csv()
  plan <- valid_plan_workbook(data, outputs = "staff")
  res <- dcc_run(data, plan = plan, output_dir = withr::local_tempdir())
  expect_identical(res$manifest$status, "partial_failure")
  expect_identical(res$manifest$reports$staff$status, "failed")
  expect_true(file.exists(res$manifest_path))

## Phase D Exit Criteria

- One validated report model drives all outputs.
- Staff Excel and HTML reports are readable, bilingual, and redacted by default.
- Statistical outputs contain complete tables, methods, hashes, and provenance.
- Machine outputs validate against versioned schemas and use deterministic paths.
- Cross-renderer counts and reconciliation values match exactly.
- Report failures are visible and do not corrupt cleaning artifacts.
- Full tests and `R CMD check` complete without errors or warnings.
