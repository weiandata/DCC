# DCC 1.1.0 AI Usability Implementation Plan

> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:executing-plans
> task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Give general-purpose AI callers a machine-readable contract for DCC:
a versioned capability document, formal JSON Schemas for every public object,
public accessors that replace hidden-attribute reads, and an `AI_USAGE.md`
that fixes the safe validate-before-execute flow. All additions are additive;
no 1.0.x call changes.

**Architecture:** `R/capabilities.R` owns `dcc_capabilities()` and its
single source-of-truth table. `R/accessors.R` owns the public accessors.
`inst/schemas/*.json` holds the published JSON Schemas, surfaced by
`dcc_schema()`. `AI_USAGE.md` documents the flow. Tests assert the capability
table matches the design matrix, the schemas validate real package output, and
each accessor equals the attribute it replaces.

**Tech Stack:** R (>= 4.1), data.table, jsonlite (schema round-trip and, in
tests, a lightweight JSON Schema check), testthat edition 3, hand-maintained
Rd documentation and NAMESPACE.

## Global Constraints

- Work on a `codex/v1.1.0-ai-usability` branch; never stage `.codegraph/`.
- Additive only: keep every existing export, column, and signature.
- Every new export gets a hand-written `man/*.Rd` and a NAMESPACE `export()`.
- New data.table column symbols go into `utils::globalVariables()`; qualify
  non-base functions (`stats::`, `tools::`).
- Schemas live under `inst/schemas/` so they install with the package; add
  `AI_USAGE.md` to `.Rbuildignore` (or place it under `inst/`).
- Focused tests run `pkgload::load_all(".", quiet = TRUE)` first; the full
  suite must stay 0 failure / 0 warning / 0 skip.

### Task 1: `dcc_capabilities()` and its source-of-truth table

**Files:** Create `R/capabilities.R`, `tests/testthat/test-capabilities.R`,
`man/dcc_capabilities.Rd`; modify `NAMESPACE`, `R/dcc-package.R`.

**Interfaces:**

- Produces: `dcc_capabilities()` -> a list with `contract_version` (a string
  such as `"1.1"`), `package_version`, and data.frames `features`
  (`name`, `status`, `since`), `rule_types`, `action_types`, and `formats`,
  plus an `unsupported` character vector.

- [ ] **Step 1 (red):** Write `test-capabilities.R` asserting the return has
  the documented names, that `features$status` is a subset of
  `c("Stable", "Experimental", "Planned")`, that `rule_types` contains all
  eight detector/rule types, that `action_types` is exactly
  `c("exclude", "set_na", "flag", "recode")`, and that `formats` lists the
  nine `dcc_read()` formats. Run and confirm failure (function missing).
- [ ] **Step 2 (green):** Implement `dcc_capabilities()` returning the table
  from the design matrix as static, deterministic data. Add
  `contract_version <- "1.1"`.
- [ ] **Step 3:** Add a *consistency* test that fails if `action_types` and the
  action names accepted by `dcc_execute()`'s validator diverge, and if
  `formats` and `infer_format()`'s switch diverge. This keeps the document
  honest as code changes.
- [ ] **Step 4:** Document `dcc_capabilities()` in Rd, export it, and add any
  new column symbols to `globalVariables()`.
- [ ] **Step 5:** Focused tests green; commit
  `feat(ai): add dcc_capabilities() capability document`.

### Task 2: Public accessors replacing hidden attributes

**Files:** Create `R/accessors.R`, `tests/testthat/test-accessors.R`,
`man/dcc_unhandled.Rd`, `man/dcc_item_map.Rd`, `man/dcc_mapping_findings.Rd`;
modify `NAMESPACE`.

**Interfaces:**

- Produces: `dcc_unhandled(x)` -> the `dcc_findings` in `x$unhandled` for a
  `dcc_result`.
- Produces: `dcc_item_map(x)` -> `attr(x, "dcc_item_map")` for a
  `dcc_map_forms()` result, with a typed error if absent.
- Produces: `dcc_mapping_findings(x)` -> `attr(x, "dcc_findings")` (the mapping
  problems) for a `dcc_map_forms()` result.

- [ ] **Step 1 (red):** In `test-accessors.R`, build a `dcc_execute()` result
  with an unhandled finding and assert `dcc_unhandled(res)$finding_id` equals
  `res$unhandled$finding_id`; build a `dcc_map_forms()` result and assert
  `dcc_item_map()` / `dcc_mapping_findings()` equal the current attributes.
  Confirm failure.
- [ ] **Step 2 (green):** Implement the accessors with `inherits()` guards and
  `dcc_type_error` on the wrong object; keep the attributes for one release for
  backward compatibility.
- [ ] **Step 3:** Rd + NAMESPACE exports; a test asserts each accessor errors
  on a wrong-class input.
- [ ] **Step 4:** Commit `feat(ai): add public result accessors`.

### Task 3: Published JSON Schemas and `dcc_schema()`

**Files:** Create `inst/schemas/finding.schema.json`,
`audit-log.schema.json`, `rules.schema.json`, `actions.schema.json`,
`manifest.schema.json`; create `R/schema.R`,
`tests/testthat/test-schema.R`, `man/dcc_schema.Rd`; modify `NAMESPACE`,
`DESCRIPTION` (Suggests: `jsonvalidate` for the test).

**Interfaces:**

- Produces: `dcc_schema(name)` -> the parsed schema (list) for
  `name %in% c("finding", "audit_log", "rules", "actions", "manifest")`, or the
  file path with `as = "path"`.

- [ ] **Step 1 (red):** Write each schema as JSON Schema draft-07 describing the
  exact fields the package produces (e.g. `finding` requires `finding_id`,
  `record_id`, `variable`, `check_id`, `evidence`, `severity`, `dimension`).
- [ ] **Step 2 (red):** In `test-schema.R`, generate a real `dcc_findings`,
  audit log, rule set (parsed YAML), action map, and manifest, convert to JSON
  with `jsonlite`, and assert each validates against its schema with
  `jsonvalidate::json_validate(..., engine = "ajv")`. Also assert a
  deliberately malformed object *fails*. Confirm the schemas exist and the
  test drives their correctness.
- [ ] **Step 3 (green):** Implement `dcc_schema()` reading from
  `system.file("schemas", ...)`; skip the validation test when `jsonvalidate`
  is absent, but require it in CI.
- [ ] **Step 4:** Rd + NAMESPACE; commit
  `feat(ai): publish JSON Schemas and dcc_schema()`.

**Files:** Create `AI_USAGE.md`; modify `.Rbuildignore`, `README.md`,

- [ ] **Step 4:** Commit `docs(ai): add AI_USAGE.md and 1.1.0 AI contract`.

## Acceptance

- `dcc_capabilities()` matches the design matrix; a test fails on divergence.
- Every schema validates a real package object and rejects a malformed one.
- Every accessor equals the attribute it replaces and errors on wrong input.
- `AI_USAGE.md` documents only implemented, exported functions.
- Package checks and tests are 0/0/0 with no skips.
