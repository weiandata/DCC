# DCC 1.1.0 Functionality Alignment Implementation Plan

> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:executing-plans
> task-by-task. Steps use checkbox syntax for tracking.

**Goal:** Close the gap between the documented design and the implementation for
the two recommended Phase 3 capabilities: a declarative codebook applicator and
declarative skip logic that feeds the missing-items detector. Decide
`dcc_to_irtc()` explicitly: implement it with a tested contract or mark it
Planned and remove it from Stable claims. No high-risk statistical action is
added.

**Architecture:** `R/codebook.R` owns `dcc_apply_codebook()` with a dry-run
preview. `R/skip_logic.R` (or an addition to `R/rules.R`) evaluates the new
`skip_logic` rule type into a structural-missingness map; `R/detectors.R`'s
`detect_missing_items()` consumes that map so a skipped item is *not
administered* rather than *omitted*. Provenance records both operations.

**Tech Stack:** R (>= 4.1), data.table, yaml, testthat edition 3,
hand-maintained Rd and NAMESPACE.

## Global Constraints

- Work on a `codex/v1.1.0-functionality-alignment` branch; never stage
  `.codegraph/`.
- Additive only; the default detector behaviour is unchanged when no
  `skip_logic` rule and no codebook are supplied.
- Raw input is never overwritten; `dcc_apply_codebook()` returns a new
  `dcc_data` version.
- Distinguish *not administered* from *omitted* explicitly in the missingness
  map and in scoring/report provenance.
- New exports get Rd + NAMESPACE; new column symbols go into
  `globalVariables()`.
- Full suite stays 0 failure / 0 warning / 0 skip.

### Task 1: `dcc_apply_codebook()` with dry run

**Files:** Create `R/codebook.R`, `tests/testthat/test-codebook.R`,
`man/dcc_apply_codebook.Rd`; modify `NAMESPACE`.

**Interfaces:**

- Consumes: a `dcc_data` (or data.frame) and a `codebook`: a list or parsed
  YAML/data.frame describing per-variable `rename`, `recode` (old -> new value
  map), `label`, `value_labels`, `missing` (values to declare missing), `type`
  (required class), and `role` (`id`/`group`/`weight`/`item`).
- Produces: with `dry_run = TRUE` (default) a `dcc_codebook_preview` listing
  every change; with `dry_run = FALSE` a new `dcc_data` version plus a
  `codebook` provenance record.
- Errors: `dcc_codebook_error` for unknown variables, impossible type
  coercions, and recodes whose old value is absent.

- [ ] **Step 1 (red):** In `test-codebook.R`, apply a codebook that renames a
  variable, recodes a value, declares a missing code, and sets a type. Assert
  the dry run reports each change and does not mutate the input, and that
  `dry_run = FALSE` produces the transformed data with a `codebook` provenance
  stage. Assert an unknown-variable codebook raises `dcc_codebook_error`.
  Confirm failure.
- [ ] **Step 2 (green):** Implement the applicator over a copied data.table,
  reusing the value-coercion approach from `dcc_execute()`'s recode path. The
  preview and the apply share one internal planner so they can never diverge.
- [ ] **Step 3:** Rd + NAMESPACE; commit
  `feat(codebook): add dcc_apply_codebook() with dry run`.

### Task 2: Declarative `skip_logic` and structural missingness

**Files:** Modify `R/rules.R` (add the `skip_logic` type to `eval_check()` or a
sibling evaluator), `R/detectors.R` (`detect_missing_items()`), Create
`tests/testthat/test-skip-logic.R`; modify `man/dcc_detect.Rd`,
`man/dcc_detect_chunked.Rd` if the chunk-safety set changes.

**Interfaces:**

- Consumes a rule of the form:

  ```yaml
  type: skip_logic
  when:
    variable: q1
    equals: 2
  then_not_required: [q2, q3, q4]
  ```

- Produces a structural-missingness map: for each record where `when` holds,
  the `then_not_required` variables are *not administered*. The map is
  attached to the detect result via a public accessor and recorded in
  provenance.
- `detect_missing_items()` gains an optional `structural` argument (the map);
  cells marked not-administered are excluded from both the numerator and
  denominator of the missingness proportion.

- [ ] **Step 1 (red):** In `test-skip-logic.R`, build data where a respondent
  skips `q2,q3,q4` because `q1 == 2`, plus a genuinely incomplete respondent.
  Assert that without skip logic the skipper is flagged for missingness, and
  with a `skip_logic` rule the skipper is *not* flagged while the genuinely
  incomplete respondent still is. Confirm failure.
- [ ] **Step 2 (green):** Evaluate `skip_logic` into the map; thread it into
  `detect_missing_items()`. Keep the default path (no map) byte-identical to
  1.0.1. Mark `skip_logic` chunk-safety in `assert_chunk_safe()` (it is
  record-local, so chunk-safe).
- [ ] **Step 3:** Update `man/dcc_detect.Rd` and add a public accessor for the
  structural map (coordinate with the AI-usability accessors). Commit
  `feat(rules): add declarative skip_logic and structural missingness`.

- Codebook dry run and apply share one planner; apply is previewed exactly.
- Skip logic makes a legitimately skipped item not-administered, and the
  missing-items detector stops flagging it, while genuine incompleteness is
  still flagged.
- The default no-skip, no-codebook detector path is unchanged from 1.0.1.
- `dcc_to_irtc()` is either complete with a tested contract or explicitly
  Planned; no fictional Stable claim remains.
- Package checks and tests are 0/0/0 with no skips.
